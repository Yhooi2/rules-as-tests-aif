# Coverage-gap patch — §1.7 / principle 08 check citation *presence*, not citation *content*

> **Date:** 2026-05-21
> **Discovered-by:** review→research→fix cycle on [`2026-05-21-swarm-tools-research.md`](2026-05-21-swarm-tools-research.md) (commits `1db3c46`, `243bd59`).
> **Status:** gap recorded; underlying fabrications already corrected. Prevention mechanism proposed, not yet built (maintainer-owned promotion decision).
> **Folder authority:** inherits [research-patches/](./) folder-level scope (one patch per gap, append-only); scope-bound by the gap below.

## Problem

The shipped research-patch `2026-05-21-swarm-tools-research.md` was committed (`e5f9c0b`) carrying **two fabricated citations**, both surviving the scope-correction commit and the patch's own anti-theatre self-check:

- **§3.9 + AGENT.md.template:** the GitHub 2,500-repo study was cited for quantitative figures («100–150-line top band», «10–15% improvement», «human-written beats generated by ~4%»). Live WebFetch of the cited article found it is **purely qualitative** — none of those numbers appear in the source.
- **§3.8:** arXiv:2408.14007 was cited for «~1-in-5 LLM comments contain demonstrably inaccurate statements». The paper's actual finding is the **opposite-leaning** 86.5 % rated equivalent-or-superior (~13.5 % below-human) — the «1-in-5» figure is not in it.

Both citations **passed the patch's own §10 §1.7 backward-check**, which asserts «each finding traced to source evidence», and would have passed a principle-08 presence check (a URL *was* attached). They were caught only by a dedicated pass that **opened each cited source and confirmed it supports the attached claim** — not by any existing automated or self-applied gate.

## Root Cause

Principle 08 ([`packages/core/principles/08-prior-art-cited.test.ts`](../../../packages/core/principles/08-prior-art-cited.test.ts)) and the §1.7 backward-check verify that a citation is **present** (a URL / SSOT-ID / ref is attached to the claim). Neither verifies that the cited source **contains** what the claim says it does. The form (citation attached) is satisfiable while the substance (source supports claim) is absent.

This is `#discipline-theatre` ([phase-research-coverage §4](../../../.claude/rules/phase-research-coverage.md)) at the **citation-truthfulness** layer, and the operational shape of **T3** («plausible-looking finding without verification», [ai-laziness-traps §2](../../../.claude/rules/ai-laziness-traps.md)) escaping a syntactic gate. It is the same family Wave 9 addressed for `Prior-art:` trailer truthfulness — but that work did not extend to **research-patch body citations**, which is where these two fabrications lived.

Contributing: the citations were authored by research subagents and accepted into the patch without the orchestrator opening each source — the `#discipline-application-scope-blindness` sub-case (c) «unverified collaborator claims» ([phase-research-coverage §4](../../../.claude/rules/phase-research-coverage.md)). The §1.7 backward-check was written *trusting* the citations it was meant to verify.

## Solution

Fabrications corrected in `1db3c46` (patch §3.8/§3.9 + template) and `243bd59` (§10 wording realignment). This patch records the **methodology gap** the incident exposed — distinct from the content fix.

## Prevention (load-bearing)

**PRIORITY CHECK — before closing a §1.7 backward-check on any artefact making external or quantitative claims:** for a sample of cited sources (floor = the highest-risk claims: every quantitative figure + every named study/paper), **open the source and confirm it states the attached claim.** A present citation is necessary, not sufficient. «Citation attached» ≠ «citation supports claim».

Mechanism candidates (in `no-paid-llm-in-ci` order of preference):

1. **Deterministic, shippable now:** extend principle 08 to assert cited arXiv IDs / URLs **resolve** (HTTP 200 / valid arXiv ID shape) — closes dead/typo'd citations, but NOT content-fabrication (a real URL can still not contain the claim).
2. **AI-agnostic reviewer step (the actual fix for content):** a sub-agent prompt (`agents/citation-content-verifier.md`-shape) read by an active session that samples cited sources and confirms support. Session-bound under subscription — **no metered CI LLM** per [`no-paid-llm-in-ci.md`](../../../.claude/rules/no-paid-llm-in-ci.md). This is the only channel that catches fabrication, and it cannot be a CI gate.
3. **Process:** orchestrator/author opens each quantitative-claim source before accepting a research-subagent's citation into a patch — the §4(c) «verify before accepting» countermeasure, applied to citations specifically.

Not pre-built — promote on evidence. **Promotion trigger candidate:** 3 fabricated-citation incidents on shipped research-patches within 6 months → build mechanism (2) as a mandatory reviewer step. This is incident **1/3**.
